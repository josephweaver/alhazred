# Runtime Philosophy

## Overview

The 24-System separates:
- static world structure,
- and mutable runtime state.

This distinction supports:
- persistent campaigns,
- AI-mediated storytelling,
- emotional continuity,
- branching narratives,
- experimental sessions,
- and evolving world perception.

The system treats narrative runtime as a living process rather than a fixed canonical script.

---

# Repository Separation

## alhazred

The primary repository contains:
- setting lore,
- rules,
- philosophy,
- locations,
- institutions,
- canonical character archetypes,
- and relatively stable world information.

This repository represents:

- structural truth,
- design philosophy,
- and long-term setting continuity.

Content here changes relatively slowly.

---

## alhazred-state

The runtime repository contains:
- active session continuity,
- mutable inventories,
- relationship changes,
- discovered clues,
- emotional state,
- runtime journals,
- active injuries,
- temporary conditions,
- and evolving narrative artifacts.

This repository represents:

living memory.

State artifacts may be:
- incomplete,
- contradictory,
- emotionally distorted,
- inaccurate,
- corrupted,
- or uncertain.

Not all runtime information is automatically canon.

---

# Canon vs Runtime

A critical distinction exists between:
- what is true,
- what characters believe,
- what players recorded,
- and what the runtime currently reflects.

The runtime layer intentionally allows:
- uncertainty,
- unreliable narration,
- fragmented memory,
- conflicting survivor accounts,
- and unstable interpretation.

Occasionally runtime discoveries may be manually promoted into canonical setting lore.

This process should remain intentional.

---

# AI-Mediated Runtime

The AI Guide functions primarily as:
- continuity support,
- atmospheric reinforcement,
- procedural arbitration,
- memory assistance,
- and narrative orchestration.

The AI should not dominate:
- player interpretation,
- emotional ownership,
- or collaborative storytelling.

Human players remain:
- meaning-makers,
- interpreters,
- emotional participants,
- and creative collaborators.

---

# Emotional Continuity

The runtime philosophy prioritizes:
- emotional persistence,
- relationships,
- ritual behavior,
- survival adaptation,
- memory,
- and atmosphere.

Narrative weight emerges through:
- continuity,
- accumulated experience,
- shared hardship,
- and evolving human connection.

The system intentionally avoids reducing emotional experience to purely mechanical state.

---

# Mechanics Philosophy

Mechanics exist primarily to:
- resolve uncertainty,
- externalize pressure,
- preserve fairness,
- and stabilize pacing.

Strong roleplay, atmosphere, and shared understanding should often resolve situations without requiring formal mechanics.

Rolls are most valuable when:
- outcomes are contested,
- uncertainty matters,
- pressure escalates,
- or meaningful consequences emerge.

---

# Runtime State as Narrative Artifact

Runtime state should be treated as:
- evolving memory,
- collaborative storytelling residue,
- and operational continuity.

The runtime is not merely bookkeeping.

It is part of the narrative experience itself.
