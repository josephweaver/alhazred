# Layered Operational Authority

## Overview

The 24-System and Alhazred runtime use layered authority structures to separate:
- stable philosophy,
- canonical setting information,
- mutable runtime continuity,
- and subjective player experience.

Different layers evolve at different rates and possess different levels of authority.

This structure helps preserve:
- coherence,
- continuity,
- ambiguity,
- experimentation,
- and long-term narrative stability.

---

# Authority Layers

## Layer 1 - Core Philosophy

Highest stability.

Examples:
- design philosophy,
- emotional principles,
- AI Guide behavior,
- runtime philosophy,
- mechanical philosophy,
- and foundational system assumptions.

Repositories:
- 24-system-core
- portions of alhazred philosophy documentation

Characteristics:
- changes slowly,
- requires intentional review,
- strongly protected from runtime drift,
- defines system identity.

The AI should behave conservatively when modifying this layer.

---

## Layer 2 - Canonical Setting

Stable but evolving.

Examples:
- Covenholm,
- Blackthorn University,
- major factions,
- core NPC archetypes,
- established locations,
- historical events,
- and long-term lore.

Repository:
- alhazred

Characteristics:
- relatively stable,
- curated intentionally,
- may occasionally absorb runtime discoveries,
- establishes shared world continuity.

Promotion into canon should remain deliberate.

---

## Layer 3 - Runtime State

High mutation layer.

Examples:
- session continuity,
- injuries,
- inventory changes,
- emotional state,
- discovered clues,
- evolving relationships,
- temporary environmental conditions,
- and active investigations.

Repository:
- alhazred-state

Characteristics:
- highly mutable,
- session-driven,
- partially subjective,
- emotionally reactive,
- may contain contradictions,
- and evolves rapidly.

This layer represents:

living memory.

The AI may update this layer aggressively.

---

## Layer 4 - Player Artifacts

Lowest authority but highest subjectivity.

Examples:
- journals,
- sketches,
- maps,
- survivor notes,
- emotional interpretations,
- rumors,
- and fragmented observations.

Characteristics:
- potentially unreliable,
- emotionally distorted,
- incomplete,
- contradictory,
- or symbolic.

Player artifacts are valuable precisely because they preserve human perspective.

They should not be forced into canonical truth automatically.

---

# Runtime Override Principle

Runtime state may temporarily override canonical assumptions operationally.

Example:

Canonical NPC:
- emotionally guarded.

Runtime state:
- currently trusts Elias Thorne.

This does not invalidate canon.

It reflects evolving lived experience.

---

# Canon Promotion

Promotion from:
- runtime state,
- session continuity,
- or player artifacts

into canonical lore should always remain:
- intentional,
- reviewed,
- and manually curated.

Not every runtime event becomes permanent world truth.

---

# AI Operational Guidance

## Conservative AI Behavior

The AI should behave conservatively when interacting with:
- core philosophy,
- foundational rules,
- or canonical setting identity.

These layers define long-term coherence.

---

## Permissive AI Behavior

The AI may behave more permissively when updating:
- runtime continuity,
- evolving relationships,
- session artifacts,
- or temporary conditions.

Controlled runtime chaos often strengthens narrative realism.

---

# Final Principle

The system intentionally separates:
- truth,
- memory,
- perception,
- canon,
- and lived experience.

Narrative meaning often emerges from the tension between them.
